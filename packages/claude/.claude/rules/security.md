# セキュリティ

## シークレット管理

- API キー・トークン・パスワードをコードに直接記述しない
- `.env` ファイルや環境変数経由で注入する
- シークレットを含むファイルは `.gitignore` に追加する
- コミット前に `git diff --cached` でシークレットの混入を確認する

## スキル内認証情報ハンドリング原則

Claude Code スキル（`~/.claude/skills/*`）で API トークン・API キー等を扱う際の原則。トランスクリプトや transcript ログへの平文露出を防ぐことが目的。

### 優先順位（上ほど推奨）

1. **MCPサーバー経由** — サーバーが認証情報を保持し、Claude本体（ハーネス）には一切渡さない。最も安全（例: `mcp__slack__*`, `mcp__github__*`）
2. **Pythonサブプロセス経由** — スキル内 `.py` が `.env` を直読みし、`requests` 等で完結させる。Bashコマンドライン層を経由しないため、変数参照すら transcript に残らない（例: `secretary/scripts/gather.py`）
3. **Bash `$VAR` 参照のみ** — `curl "...?apiKey=$BACKLOG_API_KEY"` のように変数参照だけをコマンドに書く。値は shell 内で展開され transcript には `$VAR` しか残らない。ただし事故リスクあり（下記禁止事項）

### 禁止事項（hook で強制）

以下は `validate-bash.sh` hook でブロックされる:

- **`echo $*TOKEN` / `printf $*KEY`** — 変数値の標準出力は transcript に平文で焼き付く
- **`curl -v` / `--trace` / `--trace-ascii`** — リクエストヘッダ・URLが詳細ログ出力され露出する
- **`set -x` / `bash -x`** — 変数展開値が全てトレース出力される
- **`printenv` / `env`（単体実行）** — 環境変数一覧の表示は ask 判定

### スキル作成時のチェックリスト

- [ ] 認証情報は `~/.claude/.env` に集約、ハードコード禁止
- [ ] 可能なら MCP サーバーまたは Python サブプロセスに閉じ込める
- [ ] Bash で使う場合、`$VAR` 参照のみ（値を echo/printf/cat しない）
- [ ] デバッグ時も `curl -v` / `set -x` を使わない（代わりに `curl -w "%{http_code}"` 等で必要情報だけ出力）
- [ ] エラー時の fallback メッセージに変数値を含めない

## 入力バリデーション

- ユーザー入力は必ずバリデーションする（型・長さ・形式）
- SQL クエリにはプレースホルダ（プリペアドステートメント）を使用する
- HTML 出力時はエスケープ処理を行う（XSS 防止）
- ファイルパスの入力にはディレクトリトラバーサル対策を行う

## エラーハンドリング

- エラーメッセージにスタックトレース・内部パス・DB スキーマを含めない
- ユーザー向けエラーと開発者向けログを分離する
- 認証エラーでは「ユーザー名が間違っています」等の詳細を返さない（情報漏洩防止）

## 認証・認可

- 認証トークンの有効期限を設定する
- 権限チェックはコントローラ層で必ず実施する
- CSRF トークンを適用する（状態変更リクエスト）

## 依存関係

- 既知の脆弱性を持つパッケージを使用しない
- `npm audit` / `composer audit` 等で定期的に確認する

## OWASP Top 10 チェックリスト

コードレビュー・実装時に以下を意識する：

1. **インジェクション** — SQL / コマンド / LDAP インジェクション対策
2. **認証の不備** — セッション管理・パスワードポリシー
3. **機密データの露出** — 暗号化・転送時の保護
4. **XXE** — XML パーサーの外部エンティティ無効化
5. **アクセス制御の不備** — 水平・垂直権限昇格の防止
6. **セキュリティ設定ミス** — デフォルト設定の見直し
7. **XSS** — 出力エスケープ・CSP ヘッダー
8. **安全でないデシリアライゼーション** — 信頼できないデータの検証
9. **脆弱性のあるコンポーネント** — 依存関係の更新
10. **不十分なログ・監視** — セキュリティイベントの記録
